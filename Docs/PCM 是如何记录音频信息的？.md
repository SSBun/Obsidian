下面用最直白的方式告诉你：PCM 到底是怎么把“现实世界的声音”变成一串数字的。

### 1. 现实中的声音 → 模拟波形
声音本质是空气振动 → 麦克风把振动变成连续变化的电压信号（模拟波形）。

```
现实声音 → 麦克风 → 连续的模拟波形（电压随时间变化）
```

### 2. PCM 的三个核心步骤：采样 → 量化 → 编码

| 步骤   | 英文            | 做什么                                   | 举例（CD音质）          |
|--------|------------------|------------------------------------------|--------------------------|
| ① 采样 | Sampling         | 每隔固定时间“拍照”一次波形高度           | 每秒拍 44,100 次（44.1kHz） |
| ② 量化 | Quantization     | 把“拍照”得到的高度用整数表示             | 用 16bit 整数（-32768 ~ +32767） |
| ③ 编码 | Encoding         | 把这些整数按顺序写成文件或内存           | 就是我们说的 PCM 数据     |

### 3. 用一个真实例子说明（单声道）

假设你在录一句“喂”，麦克风电压波形长这样（简化成10个采样点）：

| 时间点     | 模拟电压（真实世界） | 采样后电压 | 量化成16bit整数（PCM值） |
|------------|----------------------|------------|--------------------------|
| 0.000000s  | 0.00V                | 0.00V      | 0                        |
| 0.000023s  | +0.31V               | +0.31V     | +10137                   |
| 0.000045s  | +0.68V               | +0.68V     | +22259                   |
| 0.000068s  | +0.92V               | +0.92V     | +30146                   |
| 0.000090s  | +0.45V               | +0.45V     | +14763                   |
| ...        | ...                  | ...        | ...                      |

最终 PCM 数据就是这串整数按时间顺序排列：

```
0, 10137, 22259, 30146, 14763, -5123, -12345, -24567, -18000, 8000 ...
```

这就是原始 PCM 数据（raw PCM），没有任何压缩。

### 4. 常见的 PCM 参数（决定了音质和文件大小）

| 参数           | 常见取值                         | 说明                                      | CD音质       | 电话音质    |
|----------------|----------------------------------|-------------------------------------------|--------------|-------------|
| 采样率（Sample Rate） | 8kHz、16kHz、44.1kHz、48kHz、96kHz、192kHz | 每秒采样次数，决定最高能录多高的频率      | 44.1kHz      | 8kHz        |
| 位深（Bit Depth）     | 8bit、16bit、24bit、32bit、32bit float | 每个样本用多少位表示，决定动态范围和噪底  | 16bit        | 16bit       |
| 声道数（Channels）    | 1（单声道）、2（立体声）、5.1、7.1 | 几个独立音频轨道                          | 2（立体声）  | 1           |

计算公式（每秒数据量）：
```
比特率 = 采样率 × 位深 × 声道数
```

举例：
- CD音质：44.1kHz × 16bit × 2 = 1,411,200 bps ≈ 176KB/s ≈ 10.5MB/分钟
- 电话：8kHz × 16bit × 1 = 128,000 bps ≈ 16KB/s（比MP3还小）

### 5. PCM 数据在电脑里长什么样？

- WAV 文件 = WAV头（说明采样率、位深、声道） + 原始PCM数据
- 直播推流时 = 直接把这些整数样本塞进 AAC 编码器 → 再封装进 RTMP/FLV

### 6. 优缺点总结

| 优点                           | 缺点                         |
|--------------------------------|------------------------------|
| 完全无损，音质最高             | 文件简单，任何设备都认识 | 数据量巨大（1分钟CD音质 ≈ 10MB） 直播几乎不用裸PCM传输 |

所以现实中：
- 录音棚、音频工作站用裸 PCM（WAV、AIFF）
- 直播、视频会议一定是“PCM → 实时编码成 AAC/Opus → 网络传输”

一句话总结：
**PCM 就是把声音波形每秒拍几万次快照，然后把每张快照的高度用整数记下来，按时间顺序排好队，就成了数字音频最原始的样子。**
